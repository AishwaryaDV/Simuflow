### 2. Bug fix: `resetHistory()` incomplete state reset

`resetHistory()` (called by `clearCanvas` and `loadTopology`) cleared the undo/redo stacks but did NOT reset `_batchBase` and `_lastPushTime`. This meant the 300ms batch window could leak across clear/load boundaries — the first action after clearing could silently batch with the ghost of the last pre-clear action.

**Fix:** Added `this._batchBase = null` and `this._lastPushTime = 0` to `resetHistory()` in `GraphStore.ts`.

### 3. Bug fix: TypeScript build failure (broke Vercel deploy)

The `_snap()` method in `GraphStore.ts` creates shallow copies via object spread. TypeScript couldn't narrow the discriminated union `SimNode` back through the spread, causing `tsc -b` to fail — which broke the Vercel production build at commits `b2e7bbe` and `2fa3c79`.

**Fix:** Added `as SimNode` and `as StructuralNode` casts in `_snap()`.

### 4. Excluded test files from production build

`tsconfig.app.json` included all of `src/`, which would cause `tsc -b` to compile test files (with their intentional `as any` casts) during Vercel builds.

**Fix:** Added `"exclude": ["src/__tests__"]` to `tsconfig.app.json`.

### 5. Added `test` script to package.json

`"test": "vitest run"` — runs all tests in CI-friendly single-run mode.
